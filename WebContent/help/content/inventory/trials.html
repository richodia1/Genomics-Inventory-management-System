Trials and Traits

Managing Inventory data using Excel files introduces several problems especially when dealing with repeated measures and other related data. In Excel, this can either be resolved by adding columns to spreadsheet (e.g. Viability1, Viability2, Viability3) or using additional spreadsheets where that information can be stored in long format (e.g. Lot ID, Date, Viability).

Trial profile: Viability test

The introduction of Trials to Genebank Inventory system provides a facility to manage, store and retrieve such data measured.

Trials or Experiments

Managing germplasm requires measurement of germplasm properties (traits). Example traits: Germination rate, Viability, Moisture content, Virus indexing, etc. Measuring those properties is usually done in bulk where several lots are observed for same set of traits.

We define a Trial as a list of lots being observed and which properties are measured.

Viability testing

Seeds in Long term storage have their viability tested at several year intervals. This allows curator to establish the trends of viability within a particular crop and provides valuable insight in how viability decreases over time. To analyze these trends, all viability testing results must be available to curator. The Inventory system now provides means to hold such data.

Viability is usually measured in the same experiment as Germination rate (6 days after planting).

Trait sets

Trait sets in the system allow genebank managers to “group” individual traits into commonly used sets which are observed in the same Trial. From example above, Viability testing requires observation of two traits: Germination6d and final Viability.

Traits

A trait defines observation and measurement method used to capture germplasm properties. Observing the same property but with different measurement method calls for new trait.

Trait conversion

Different traits measuring the same property can sometimes be converted or merged into one. Measuring 100 seed weight in milligram instead of gram allows for conversion between units as long as the method of observation is exactly the same.

Trials in Inventory system

Using Inventory system to capture trial data follows:

  1. Establish traits
  2. Create trait set
  3. Create Trial and select trait set
  4. Select the lots that should be observed in the Trial
  5. Enter captured data

Steps 1 and 2 above are system “configuration” steps, while steps 3-5 are daily use steps.

Configuring Trials system

To convert existing practices to Traits system, managers should review existing practices in reverse order:

  1. Review data and data types captured in previous trials
  2. Review and document lot selection process
  3. Identify trial properties (date, naming)
  4. Identify set of traits observed in trial
  5. Identify and document individual traits

Establishing traits

An individual Trait captures the following information: Trait name, Unique short name (XLS variable name), Method of observation and measurement, Data type and scale.

Coded traits

Observed data is usually captured in numeric form (100%, 44%, 1.3g, etc). Using coding information, one can define a fixed set of values that can be used as trait values. Measuring susceptibility to virus is usually captured in “text” form: Positive +++, Positive ++, Positive +, Marginal +/-, Negative -.

The Traits system allows to store these as “Coded” traits where unique values are assigned to all coded values: 3 for Positive +++, 0 for Marignal, -1 for Negative.

Example coded trait

Using coded traits limits what data can be entered as observation results.

Creating trait sets

In a single Trial several different traits may be observed. To simplify Trial configuration, you can register Trait sets that are commonly used. A trait set “Viability testing” with Germination6d and final Viability is an example of trait set.

Note, there is no limit on which traits are measured in a single Trial.

Using Trials system

Trials system captures data which lots are being tested, when the trial started, trial title and description and most importantly, which Traits are observed.

Creating Trials

Clicking “Create new trial” on dashboard opens Trial properties form:

Trial properties

  • Trial name: give meaningful name to trial
  • Date of trial
  • Trait group: select group of traits that are observed in the trial.
  • Description: additional description of trial.

Once trial is record is created, register lots with the trial. This records that a particular lot should be included in trial and has to be included in observation.

Selecting lots

Registering lots to be observed in a trial is done by entering Lot ID numbers or Lot barcode numbers (using barcode reader).

Note: Trials can also quickly be created for any committed Inventory update record (e.g. Regeneration) by clicking the “Create trial” button. This will register all lots in the update for trial.
